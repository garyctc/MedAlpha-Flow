Project: MedAlpha Connect (MVP V1)
Short Description
A smartphone app is to be created for "MedAlpha". This app will be utilized within the framework of MedAlpha retail partner projects. It will not be used in the context of cannabis-related projects.
Functional Overview
	•	"Home screen" displaying the app's features and capabilities, including dynamic content management (CMS)
	•	User management including registration
	•	Profile/Settings
	•	Two main functions:
	•	Appointment booking (Primary feature – key for the user)
	•	Direct, binding doctor's appointment booking via the "Smart Match" service
	•	Telemedicine appointment (video consultation; via Teleclinic)
	•	Prescription redemption (Secondary offering)
	•	Online: E-prescription (public and private health insurance) redemption at the "Apo Group" online pharmacy via Cardlink
	•	Offline: Pharmacy search (Google Maps-style)
	•	History (past prescription redemptions via Cardlink, appointment bookings, and telemedicine consultations)
	•	Additional static pages for FAQ, Support, Privacy Policy, and Legal Disclosure
	•	Push notifications
Further Assumptions
	•	Modern and sleek design with intuitive user navigation.
	•	Access to the two primary features is only possible with a completed profile.
	•	Later in the project, cooperation partners will "hand over" their customers to the app via SSO. This must be accounted for in the initial planning.
	•	The entire app must be built from the ground up to allow identical functionality to be released under a different name and design (maintaining the same release cycle).
	•	Users must be able to choose between "online" and "offline" pharmacies before redeeming a prescription (legal requirement). If "offline" is selected, a pharmacy search will be provided. The vision is to integrate a Google Maps search based on the user's location.
	•	Push notifications will be used to alert users to new offers.
	•	Successful appointment bookings will be stored in the user's history. Once the appointment has concluded (60 minutes after the scheduled end), a notification will be sent to the user. This notification should ask if everything was satisfactory and whether they would like to redeem a prescription via the app or require a pharmacy recommendation.
German Version
Projekt: MedAlpha Connect (MVP V1)
Kurzbeschreibung:
Es soll eine Smartphone-App für „MedAlpha“ erstellt werden. Diese App wird im Rahmen der MedAlpha-Retail-Partner-Projekte genutzt. Diese App wird nicht im Rahmen der Cannabis-Projekte verwendet.
Funktionsübersicht:
	•	„Startseite“ auf der die Möglichkeiten der App dargestellt werden inkl. dynamischer Anpassung (CMS)
	•	Benutzerverwaltung inkl. Registrierung
	•	Profil/Einstellungen
	•	Zwei Hauptfunktionen:
	•	Terminbuchung (Hauptfunktion – wichtig für den Nutzer)
	•	Direkte verbindliche Arztterminbuchung mit dem Dienst „Smart Match“
	•	Telemedizintermin (Videosprechstunde; über Teleclinic)
	•	Rezept-Einlösung (sekundär Angebot)
	•	Online: eRezept (GKV und PKV) Einlösung bei der „Apo Group“ Online-Apotheke mit Cardlink
	•	Offline: Apothekensuche (Google Maps like)
	•	Historie (vergangene Rezepteinlösungen mit Cardlink, Terminbuchungen und Inanspruchnahme von Telemedizin)
	•	Weitere statische Seiten für FAQ, Support, Datenschutz und Impressum
	•	Push-Notification
Weitere Annahmen:
	•	Modernes und simples Design mit einfacher Benutzerführung
	•	Die Nutzung der zwei Hauptfunktionen ist nur mit einem vollständigen Profil möglich.
	•	Es wird im späteren Verlauf des Projektes Kooperationspartner geben, die über „SSO“ ihre Kunden an diese App „übergeben“. Das soll schon im Voraus mitberücksichtigt werden.
	•	Die gesamte App muss von vornherein so gebaut sein, dass wir die Identischen Funktionen mit einem anderen Namen und Design veröffentlichen können (gleicher Release Zyklus)
	•	Der Nutzer muss vor der Rezepteinlösung die Möglichkeit haben, zwischen „online“ und „offline“ Apotheke zu unterscheiden (gesetzliche Vorgabe). Bei der „offline“ Auswahl, wird eine Apothekensuche zur Verfügung gestellt. Die Vorstellung ist, dass eine „Google Maps“ Suche integriert ist und abhängig vom Standort des Nutzers gesucht wird.
	•	Die Push-Notifications sollen dazu verwendet werden, um den Nutzer auf neue Angebote aufmerksam zu machen.
	•	Eine erfolgreiche Terminbuchung wird in der Historie für den Nutzer gespeichert. Wenn der Termin stattgefunden hat (60 Minuten nach geplantem Ende) wird eine Notification an den Nutzer geschickt. Diese Notification soll den Nutzer fragen ob alles in Ordnung war und ob er ein Rezept mit der App einlösen möchte, bzw. ob eine Apotheke vorgeschlagen werden soll.
